# Design Drift Routine

Run this for UI-heavy products.

## Checks

1. UI components that do not map to documented patterns.
2. Implemented tokens that diverge from `wiki/design-system/tokens.md`.
3. Accessibility rules without tests or manual evidence.
4. Interface copy that uses undefined glossary terms.

## Output

- Design wiki updates.
- UI consistency issues.
- Checks or stories needed.
